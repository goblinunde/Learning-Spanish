# JSON-to-TeX 转换器设计规格

## 1. 目标

为 SpanishCards 增加离线 JSON-to-TeX 转换工作流，使词库可以在编译前转换为普通 `.tex` 文件。转换器必须同时支持：

- 可由 `\input` 引入的词条片段。
- 包含 `\documentclass` 的完整独立文档。
- 完整文档选择卡片或表格输出。
- 单文件转换与 `data/*.json` 批量转换。
- 通过 Makefile 提供易记、可发现的命令。
- 与运行时 JSON 导入使用相同的解析、字段映射和 TeX 转义规则。

## 2. 非目标

- 不加入 JSON 以外的新数据格式转换器。
- 不在转换期间自动翻译、生成例句或联网查询。
- 不在生成文件中执行 JSON 提供的 LaTeX 命令。
- 不引入 Python、Node.js、jq 或第三方 Lua 库。
- 不把生成文件作为必须提交的源文件；用户可以随时由 JSON 重建。

## 3. 文件结构

新增文件：

```text
Makefile
scripts/json-to-tex.lua
tests/test-generated-fragment.tex
```

修改文件：

```text
spanishcards-data.lua
tests/run-tests.sh
README.md
```

运行后生成：

```text
generated/fragments/<name>.tex
generated/documents/<name>.tex
```

## 4. 转换器命令行接口

底层命令：

```bash
texlua scripts/json-to-tex.lua \
  --input data/example.json \
  --output generated/example.tex \
  --format fragment \
  --view cards
```

参数：

| 参数 | 必填 | 允许值 | 默认值 |
|---|---:|---|---|
| `--input` | 是 | 可读 JSON 文件 | 无 |
| `--output` | 是 | 可写 `.tex` 路径 | 无 |
| `--format` | 否 | `fragment`、`document` | `fragment` |
| `--view` | 否 | `cards`、`table` | `cards` |
| `--help` | 否 | 无值 | 无 |

`--view` 只影响 `document` 格式；片段格式接受该参数但不写入打印命令。

非法格式、非法视图、缺少参数、输入不存在、JSON 错误或输出不可写时，转换器向标准错误输出清晰信息并以非零状态退出。

## 5. 片段格式

片段只包含说明注释和词条命令：

```latex
% Generated by SpanishCards. Do not edit manually.
% Source: data/example.json

\AddSpanishWord{spanish={...},english={...}}
\AddSpanishWord{spanish={...},english={...}}
```

使用方式：

```latex
\input{generated/fragments/example.tex}
\PrintVocabularyCards
```

生成注释不包含时间戳，使相同输入得到字节稳定的输出。

## 6. 完整文档格式

卡片视图：

```latex
\documentclass{spanishcards}

\begin{document}
\AddSpanishWord{...}
\PrintVocabularyCards
\end{document}
```

表格视图将打印命令替换为 `\PrintVocabularyTable`。完整文档不额外加载外部 JSON，因此生成后可独立于原始数据文件编译；仍需同目录或 TeX 路径中存在 SpanishCards 模块。

## 7. Makefile 接口

```bash
make help
make test
make pdf
make convert-json INPUT=data/example.json \
  OUTPUT=generated/custom.tex FORMAT=fragment VIEW=cards
make json-fragments
make json-documents
make clean
make distclean
```

目标行为：

- `help`：列出目标、变量、默认值和示例。
- `test`：运行 `./tests/run-tests.sh all`。
- `pdf`：使用 LuaLaTeX 编译 `example.tex` 并更新根目录 `example.pdf`。
- `convert-json`：按 `INPUT`、`OUTPUT`、`FORMAT`、`VIEW` 转换一个文件。
- `json-fragments`：将所有 `data/*.json` 转换为 `generated/fragments/*.tex`。
- `json-documents`：将所有 `data/*.json` 转换为 `generated/documents/*.tex`，默认卡片视图。
- `clean`：删除 `build/`、`generated/` 与根目录 LaTeX 临时文件，但保留 `example.pdf`。
- `distclean`：执行 `clean` 后再删除 `example.pdf`。

Makefile 设置工作区内的 `TEXMFVAR` 与 `TEXMFCACHE`，避免受限环境中的字体缓存写入问题，同时允许用户通过环境变量覆盖。

## 8. 共享数据模块重构

`spanishcards-data.lua` 新增纯函数导出：

- `parse_json(content)`：解析字符串并返回根值。
- `read_json_records(path)`：读取并验证对象数组。
- `record_to_latex(record, source, index)`：返回一条安全的 `\AddSpanishWord` 文本。
- `records_to_latex(records, source)`：返回全部词条文本。

现有 `load_json` 改为调用这些纯函数后通过 `tex.sprint` 输出。CSV 行为不变。转换器只调用纯函数，不复制 JSON 解析器或 TeX 转义代码。

## 9. 输出与写入规则

- 输入和输出均使用 UTF-8。
- 输出目录不存在时自动创建。
- 先写同目录临时文件，再重命名到最终路径，避免失败时留下半个文件。
- 外部文本继续转义反斜杠、花括号、百分号、`&`、`#`、下划线、波浪号和尖号。
- JSON 未知字段被忽略并产生警告。
- 缺少 `spanish` 的记录被跳过并产生警告，不写入生成文件。
- 记录顺序与 JSON 数组顺序一致，字段顺序与现有 `field_order` 一致。

## 10. 测试与验收

自动测试必须验证：

1. 转换器不存在时测试先失败。
2. 片段输出包含自动生成标头和 `\AddSpanishWord`，不包含 `\documentclass`。
3. 片段可由测试文档 `\input` 并成功生成两页卡片。
4. `document + cards` 输出包含文档类和 `\PrintVocabularyCards`，可成功编译。
5. `document + table` 输出包含 `\PrintVocabularyTable`，可成功编译。
6. Unicode、转义字符、数字、布尔值和 `null` 与运行时导入结果一致。
7. 相同输入连续转换两次得到相同文件哈希。
8. `make help`、`make convert-json`、`make json-fragments` 和 `make json-documents` 成功。
9. `make test` 运行全部测试。
10. `make pdf` 生成 16:10 的 `example.pdf`，日志无错误、溢出和缺字。

## 12. 自审结论

- 片段与完整文档的边界明确，完整文档还可选择卡片或表格。
- CLI 与 Make 两层接口分离，方便自动化和直接使用。
- 解析与转义逻辑只保留一份，避免运行时导入和预生成结果漂移。
- 输出无时间戳且采用原子写入，适合增量构建和版本比较。
- 所有功能保持离线、自包含并继续只依赖 TeX Live。
